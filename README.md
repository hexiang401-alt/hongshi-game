# 红十多人房间版 v3.1

这是一个 6 人多人房间版网页游戏，后端使用 Python 标准库 `http.server`，前端入口是 `web/index.html`。当前项目不依赖第三方 Python 库。

## 本地运行

在仓库目录下运行：

```bash
python server.py
```

本地默认端口是 `8000`。启动后房主打开：

```text
http://127.0.0.1:8000
```

同一局域网的其他玩家可以打开：

```text
http://房主电脑IP:8000
```

例如房主电脑 IP 是 `192.168.1.23`，其他玩家打开：

```text
http://192.168.1.23:8000
```

如果环境变量里设置了 `PORT`，服务会使用该端口；没有设置时默认使用 `8000`。

## Render 部署

在 Render 创建 Web Service，并使用以下配置：

```text
Build Command: pip install -r requirements.txt
Start Command: python server.py
```

Render 会通过环境变量 `PORT` 指定端口，`server.py` 会自动读取该端口并监听 `0.0.0.0`。

## 玩家如何开局

1. 房主打开部署后的网址或本地网址，输入昵称，点击“创建房间”。
2. 页面会显示房间码，房主把房间码发给另外 5 个玩家。
3. 其他玩家打开同一个网址，输入昵称和房间码，点击“加入房间”。
4. 满 6 人后，房主点击开始游戏。

## 已实现

- 6 人加入同一房间，真实对战。
- 房主创建房间，其他玩家输入房间码加入。
- 每个玩家只看到自己的手牌。
- 说话、出牌、不要、结算均由服务器同步。
- 出牌阶段打出红十只公开身份，不触发翻倍。
- 只有说话环节中身份被公开/揪出，底分才会翻倍。
- 玩家可以对其他未公开身份的玩家设置私人判断：未知 / 怀疑红十 / 认为平民。
- 红桃 3 先出。
- 下一局由上一局第 1 名先发牌、先说话。
- 保留出牌音效。

## 注意

当前版本使用 HTTP 轮询，不依赖第三方库，适合本地、同一 Wi-Fi 或 Render Web Service 部署测试。
